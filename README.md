# copy-space-cleaner

一个零依赖的本地小工具：

- 左侧粘贴网页/PDF复制出的原文
- 按“Word 友好”规则自动清理空格（中英贴合、数字与单位留 1 空格）
- 右侧实时生成并可继续手动编辑 Markdown
- 可覆盖保存或追加保存到本地 `md` 文件

## 1. 启动

在项目目录运行：

```bash
python3 -m http.server 5173
```

然后打开：

- <http://localhost:5173>

建议浏览器：Chrome / Edge（支持直接写入本地文件）。

## 2. 使用流程

1. 把网页或 PDF 文本粘贴到左侧“原始文本”区。
2. 右侧会实时生成清理后的 Markdown。
3. 点击“打开已有 md”读取现有文件（会自动导入并清理）。
   - 如果当前编辑区已有内容，会先弹窗确认是否替换，避免误覆盖。
4. 点击“新建/另存为 md”可指定一个新的保存目标文件。
5. 可选：
   - 开启“自动覆盖保存到 md”做实时落盘。
   - 或手动点“覆盖保存 / 追加保存”。
6. 在“规则设置（Word 友好）”里可切换规则，并修改单位列表。
7. 如浏览器不支持直接写文件，可点“下载 md”。

## 3. 默认规则（Word 友好）

- 去掉中文和中文之间空格
- 去掉中文与英文字母/数字之间空格（例如 `我是 AI` -> `我是AI`）
- 去掉中文与中文标点前后的空格（例如 `我们提出 一种方法。 日常` -> `我们提出一种方法。日常`）
- 数字与单位之间标准化为一个空格（例如 `10MB/s` -> `10 MB/s`）
- 英文段落中多余空格压缩为一个
- 去掉行尾空白
- 超过 2 个连续空行压缩为 2 个

## 4. 单位规则说明

- “数字与单位空格”只处理你在“单位列表”里配置的单位。
- 单位列表用逗号分隔，例如：`kg,km,MB/s,ms,°C,kWh`。
- 你可以按文档类型增删单位，避免误替换。

## 5. 文件结构

- `index.html`：界面
- `app.js`：规则引擎、清理与文件保存逻辑
- `README.md`：说明文档
